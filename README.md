# CS172 Web Crawler - Project Part A

## Overview

This project implements a web crawler using Python and the Scrapy framework. The crawler starts from a set of seed URLs, follows hyperlinks up to a specified depth, and collects HTML pages.

The system is designed with a modular architecture, separating responsibilities into:
- Crawling (Scrapy Spider)
- Link filtering
- Page storage
- Logging

## Setup

### Mac / Linux

```bash
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

```
### Windows

scripts\crawler.bat crawler\seed.txt 10000 3 data

### Arguments

• crawler/seed.txt → file containing seed URLs (one per line)
• 10000 → maximum number of pages to crawl
• 3 → maximum crawl depth (number of hyperlink hops from seed URLs)
• data → output directory where HTML pages will be stored

### Project Structure

crawler/

### System Behavior

• The crawler avoids duplicate pages using a visited URL set.
• Only valid HTTP/HTTPS links are followed.
• Non-HTML resources (e.g., images, PDFs, scripts) are filtered out.
• The crawler respects both a maximum page limit and depth constraint.
• Scrapy’s asynchronous architecture enables concurrent crawling.

### Limitations

• Some websites may block requests or enforce rate limits.
• Certain pages may fail due to network timeouts.
• JavaScript-rendered content is not processed (static HTML only).
• The crawler does not restrict domains unless explicitly configured.
